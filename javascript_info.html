JavaScript (JS)

Purpose

JavaScript was purpose built as the programming language of HTML and the Web (W3 Schools, 2020). It was born out of a vision from Netscape Communications founder Marc Andreesen to make the web more dynamic. Simple enough for non-programmers to pick-up while allowing interactions with the web page's Document Object Model.

Document Object Model (DOM)

The browser creates a DOM when the pages is loaded. The DOM is an object model, where each element is represented as a descendent of the encolsing element, as shown in the figure below. JavaScript is able to dynamically access and update the content and style of the web page through the DOM.

Graphical Representation of the Document Object Model
(W3 Schools, 2020)

For example, the following javascript code takes the value from an input field (id of "newHeading"), and dynamically updates the first h1 of the webpage. Try it below.


function changeHeading1() {
    var h1, newHeading;
    newHeading = document.getElementById("textInput").value; 
    h1 = document.getElementsByTagName("h1");
    h1[0].innerHTML = newHeading;
}

AJAX

Asynchronous JavaScript And XML (AJAX) is a method for updating a webpage, requesting & recieving data, and sending data, all after a page has loaded (W3 Schools, 2020). For example, when a number is submitted using the form below, JavaScript uses AJAX to request a fact about that number from www.numbersapi.com and dynamically display it on the page. The process is as follows:


  1. .addEventListener
    waits for the user to submit the form.
  2. .getElementById
    is used to obtain the form values.
  3. Concatenation is used to create a custom url with the requested number and type
  4. fetch
    is used to make the request for the external data. While the data is being requested the user can continue to interact with the webpage.
  5. Once the data is recieved, it is read in a JSON format
  6. JavaScript uses the DOM, described above, to access and dynamically update the webpage with the recieved fact.


History

Date Event
April 1995 Brenden Eich is engaged to develop Mocha, the initial name for JavaScript (Dahl, 2012). At the same time the Java language, developed by Sun Microsystems, was beginning to gain traction (Peyrott, 2017). Sun partners with Netscape to make Mocha a scripting companion for Java, and integrate into the Netscape browser. The intent is to aim Java at professional developers, and Mocha (now JavaScript) to amateurs and designers.
May 1995 There is immense internal pressure to close the deal with Sun. Therefore Brendan Eich creates a working prototype for Mocha within several weeks, and it is integrated into the Netscape browser (Peyrott, 2017). Mocha is renamed LiveScript.
Dec 1995 The deal between Netscape and Sun is closed, and the language is renamed JavaScript (Peyrott, 2017).
Aug 1996 Microsoft implements their own version of JavaScript (JScript) in Internet Explorer 3.0. Implementation of JavaScript is beginning to fragment (Peyrott, 2017).
Nov 1996 Industry association ECMA begins standardisation of JavaScript (Peyrott, 2017). The purpose of standardization is to better control the evolution of the language and prevent fragmentation. The language described by this standard would be called ECMAScript. JavaScript remains the commercial name for ECMAScript.
June 1997 The first version of ECMAScript is released (Peyrott, 2017).
June 1998 The second version of ECMAScript is released to resolve inconsistences between the ECMA and ISO standards (Peyrott, 2017).
June 1998 ECMAScript 3 is released, bringing in regular expressions, do-while blocks, exceptions and try/catch blocks, improved error handling, among other functionality (Peyrott, 2017).
June 1998 Asynchronous JavaScript and XML (AJAX) is developed as a technique to allow pages to be updated on the fly, not only as a result of a user event (Smith, 2006).
2008 After 8 years ECMAScript 4 is scrapped amongst disagreement over the direction of the language (Peyrott, 2017).
2009 ECMAScript 5 (previously known as ECMAScript 3.1) is signed off, becoming one of the most supported versions of JavaScript (Peyrott, 2017).
2015 ECMAScript 6 (later renamed ECMAScript 2015) is released, implementing some of the big features of the discarded ECMAScript 4 (Peyrott, 2017).
2016-2018 From now on ECMAScript versions are released yearly and named after the year of release (Peyrott, 2017).
